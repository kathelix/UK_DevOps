## Collector (`apps-script/gmail-collector.gs`)

- [ ] **Fetch via label store instead of search index.** The `q=`-based listing (inherited from Make) reads Gmail's search index, which silently skips unindexed messages — observed 2026-06-07 with securityclearedjobs.com emails: visible in the Gmail UI, invisible to every API query (`from:`, `subject:`, `in:anywhere`). Switch to label-store listing (`getUserLabelByName('job-vacancies')` / `labelIds`-based) to make such orphans structurally impossible. _(Parked pending the 2026-06-21 `label-store-fetch-recheck` probe.)_
- [ ] **Map `nexxt.com`'s footer (deferred — fixture-fidelity blocker).** The PII bar holds (a single `sent by Nexxt, c/o Nexxt Inc` text marker leaves no recipient email post-cut in **both** the `alert@` postal-first and `jfw@` action-first templates, byte-confirmed 2026-06-20), but a faithful raw-capture fixture is not producible: the live collector keeps the QP body **literal** while the Gmail-MCP `get_thread` **QP-decodes** it to C0 control bytes inside tracking URLs (`jfw@` is also effectively n=1). Map it once an **accepted collector-equivalent HTML-extraction path** lands — a byte-preserving raw RFC822 export *does* exist (e.g. `read_email(include_raw_mime)`, **not** in Claude Code's connector; Codex verified it returns the literal `=NN` with zero control bytes/U+FFFD), but it is **not sufficient alone**: a faithful fixture must equal the collector's extracted `htmlBody` (`extractHtmlBody_` over the decoded MIME tree). The `fixture-raw-transport` slice (2026-06-24) **built** this path (parity-gated raw-RFC822 capture) and mapped `talent.com` through it; `nexxt.com` just needs its own capture + per-message parity pass via the same transport (marker `sent by Nexxt, c/o Nexxt Inc`, `text`; `jfw@` effectively n=1). See `docs/TECH_DESIGN.md` §4 (the `nexxt.com` deferral note).
- [ ] **Second cleaning pass.** The regex strips attributes/comments/images but leaves bare tag skeletons (`<td>`, `<tr>`, `<a href>`) and undecoded entities (`&amp;`, `&pound;`).
